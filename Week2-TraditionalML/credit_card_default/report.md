# 信用卡违约预测 — 实验报告

## 1. 问题描述

基于信用卡用户的 23 个特征（信用额度、还款历史、账单金额、人口统计等），预测下个月是否发生违约。二分类任务，30,000 条数据，违约率约 22%（类别不平衡）。

## 2. 数据预处理

- 删除 ID 列（无关特征）
- 所有数值特征 `StandardScaler` 标准化
- 80/20 分层划分 (stratify=y)

## 3. 模型对比

| 模型 | Accuracy | AUC | 特点 |
|---|---|---|---|
| Logistic Regression | 81.60% | 0.7414 | 线性基准，召回率极低 |
| Decision Tree | 81.78% | 0.7278 | 可解释，易过拟合 |
| **Random Forest** | **81.67%** | **0.7727** | 集成学习，特征重要性可解释 |

## 4. 分析

- **类别不平衡问题**：三个模型对违约类（少数类）的召回率均偏低（35-40%），模型倾向于预测"不违约"以求高准确率
- **AUC 是更合适的指标**：准确率 (~82%) 看似不错，但违约类召回率仅 35%，AUC 反映了模型在区分能力上的真实水平
- **RF 最优**：AUC=0.7727，特征重要性分析显示 PAY_0（最近还款状态）是最强的预测因子——还款历史直接反映借款人当前的财务状况
- **改进方向**：SMOTE 过采样、class_weight 调整、阈值优化以提高违约类的召回率

## 5. 结论

信用卡违约预测是一个典型的类别不平衡任务。Random Forest 在 AUC (0.7727) 上表现最好，但违约类的召回率（35%）仍有很大改善空间，需要专门处理不平衡学习。
