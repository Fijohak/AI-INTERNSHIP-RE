# AI 实习 Week1-2 知识词汇表

> 这份文档写给一个「已经写出代码但还没完全理解背后概念」的实习生。
> 每个词条都附带：它是什么 + 一句话核心思想 + 推荐你问 GPT 的问题。

## 第一部分：基础概念（先看这里）

### 1. 机器学习到底是什么？

一句话：**让计算机从数据中自动找到规律，然后用这个规律做预测。**

比如你给计算机看 1000 个人的体检报告（血糖、血压、年龄...）和这些人是否得了糖尿病。计算机自己总结出一套规则，下次来一个新人的体检报告，它就能预测这个人会不会得糖尿病。

### 2. 监督学习 vs 无监督学习

- **监督学习**：训练数据有「标准答案」（标签）。比如每条数据都标注了"违约"还是"没违约"。**分类和回归都属于这一类。**
- **无监督学习**：训练数据没有标准答案。让计算机自己发现数据中的群体。**聚类和异常检测属于这一类。**

> 去问 GPT："请用一个生活中的例子，给我解释监督学习和无监督学习的区别"

### 3. 分类 vs 回归

- **分类**：预测的结果是一个**类别**。比如"这个人会不会违约？会/不会"。
- **回归**：预测的结果是一个**连续的数值**。比如"这套房子值多少钱？$350,000"。

> 去问 GPT："分类和回归的本质区别是什么？为什么分类用准确率评估，回归用 R² 评估？"

### 4. 训练集、验证集、测试集

- **训练集**：用来教模型的"课本"。
- **验证集**：用来调参的"模拟考"（边学边检查，调整学习策略）。
- **测试集**：最终考试，只能考一次。用来报告最终成绩。

我们的项目用了简化的二划分（训练+测试），没有单独分出验证集。在工业界，更严格的做法是三划分。

> 去问 GPT："什么是数据划分？训练集验证集测试集分别起什么作用？"

## 第二部分：数据预处理

### 5. StandardScaler（标准化）

**它是做什么的？** 把不同量纲的特征拉到同一个尺度上。

**为什么需要它？** 比如你的数据里，"年龄"的范围是 20-80，"收入"的范围是 20000-200000。如果不做标准化，收入这个特征因为数值大，会对模型产生不成比例的巨大影响。标准化后，所有特征的均值=0，标准差=1，模型对每个特征一视同仁。

> 去问 GPT："StandardScaler 和 MinMaxScaler 的区别是什么？什么时候用哪个？"

### 6. One-Hot Encoding（独热编码）

**它是做什么的？** 把「类别文字」变成「0 和 1 的向量」。

**例子**：有一个特征是"颜色"，取值是 红/绿/蓝。One-Hot 之后变成三个新特征：
- is_红: 1 或 0
- is_绿: 1 或 0
- is_蓝: 1 或 0

这样计算机才能做数学运算。如果直接给"红=1, 绿=2, 蓝=3"，计算机会错误地认为"蓝 > 绿 > 红"。

> 去问 GPT："什么是 One-Hot Encoding？它和 Label Encoding 有什么区别？"

### 7. 缺失值处理

真实数据几乎一定会有缺失值。常见处理方式：
- **删除**：扔掉缺失的数据（简单但有信息损失风险）
- **填充**：用中位数/均值/众数填上（最常用）
- **标记**：新增一列标记哪些值原本是缺失的（保留信息）

我们的糖尿病项目中，Glucose、BMI 等医学生理指标出现 0 值——在医学上血糖不可能为 0，所以这些 0 其实是"没测到"的意思，就是缺失值。我们用中位数填充了它们。

> 去问 GPT："数据缺失值有哪些处理方法？分别适用于什么场景？"

### 8. 数据可视化（EDA — Exploratory Data Analysis）

**EDA = 探索性数据分析**。在真正训练模型之前，先用图表观察数据长什么样，发现异常值和规律，这能帮你避免很多低级错误。

我们项目中的 EDA 方法：
- **直方图**：看一个特征的分布（数据集中在哪个范围？是正态分布还是偏态？）
- **散点图**：看两个特征之间的关系（正相关？负相关？有没有异常点？）
- **箱线图**：看数据的四分位数和异常值
- **饼图**：看类别占比（比如信用卡数据中 22% 违约）
- **热力图/相关性矩阵**：同时看所有特征两两之间的相关性

> 去问 GPT："什么是探索性数据分析（EDA）？一个完整的数据分析项目通常包含哪些 EDA 步骤？"

## 第三部分：模型 — 它们分别是什么？

### 9. Logistic Regression（逻辑回归）

**名字有误导性**：虽然叫"回归"，但它是个**分类**模型。主要用于二分类问题（是/否，会/不会）。

**核心思想**：给每个特征一个权重，加权求和后再通过一个 S 形函数（sigmoid）把结果压缩到 0-1 之间，作为"属于正类的概率"。

**优点**：训练快、可解释性强（能直接看到每个特征对结果的影响方向和大小）。

> 去问 GPT："Logistic Regression 为什么名字里有 Regression 却用来做分类？它的决策边界是什么样的？"

### 10. Decision Tree（决策树）

**核心思想**：像玩 20 问游戏一样，一层一层根据条件分支，最终到达一个叶子节点（决策结果）。

**例子**：
```
血糖 > 120? → 是 → BMI > 30? → 是 → 预测：糖尿病
           → 否 → 年龄 > 50? → 是 → 预测：有风险
                              → 否 → 预测：健康
```

**优点**：完全可解释（你可以画出整棵树），不需要特征标准化。
**缺点**：容易过拟合（树太深时会记住训练数据中的噪声）。

> 去问 GPT："决策树的过拟合是怎么发生的？为什么剪枝（pruning）能解决？"

### 11. Random Forest（随机森林）

**核心思想**：种很多棵决策树（比如 100 棵），每棵树的训练数据和特征子集都不一样，最后投票决定结果。

**为什么比单棵树好？** 就像一个决策可能出错，但 100 个人投票的结果更可靠。每棵树的角度不同，错误会互相抵消。

**优点**：几乎不需要调参就能得到不错的结果，不容易过拟合。
**附带产物**：`feature_importances_`——它能告诉你每个特征对预测的贡献有多大。

> 去问 GPT："Random Forest 为什么能避免过拟合？Bagging 和随机特征选择分别起什么作用？"

### 12. XGBoost

**核心思想**：也是一种集成树模型，但它种的树不是独立的——每棵新树专门去纠正前面所有树的错误。

**和 Random Forest 的区别**：
- RF：100 棵树同时种，各干各的，最后投票
- XGBoost：一棵一棵种，后面的树专注于修正前面的错误（像老师给你讲解错题）

**优点**：在 Kaggle 竞赛和工业界几乎是表格数据的首选模型，精度通常最高。
**缺点**：超参数多，调参比 RF 复杂。

> 去问 GPT："XGBoost 和 Random Forest 的核心区别是什么？Gradient Boosting 的'梯度'指的是什么？"

### 13. KNN（K 近邻）

**核心思想**：找离新样本最近的 K 个老样本，看它们大多数属于哪个类别，新样本就归哪类。

**优点**：极其简单直观，不需要训练过程。
**缺点**：预测慢（要和所有训练数据计算距离），对特征尺度敏感（必须先标准化）。

> 去问 GPT："KNN 中的 K 值怎么选？K 太小和太大会分别导致什么问题？"

### 14. SVM（支持向量机）

**核心思想**：找一条线/平面把不同类别的数据分开，而且这条线离最近的数据点尽可能远（最大间隔）。

用 RBF 核时，SVM 可以把原本线性不可分的数据映射到高维空间，在高维中找到分割面。

**优点**：在小数据集上表现极好（我们的 Iris 项目就是 SVM 最好）。
**缺点**：数据量大时训练很慢。

> 去问 GPT："SVM 的核函数是什么？RBF 核为什么能让 SVM 处理非线性问题？"

### 15. Isolation Forest（孤立森林 — 异常检测）

**核心思想**：异常点有个特点——它们很容易被"孤立"出来。想象在一群人中找出一个外星人，你只需要很少几个问题就能把外星人挑出来；但要在一群普通人类中挑出某一个具体的人，需要问很多问题。

Isolation Forest 建立很多棵随机的决策树，看每个样本平均用多少次划分就能被"孤立"——异常点通常只需要很少几次划分。

> 去问 GPT："Isolation Forest 的直觉原理是什么？为什么异常点更容易被孤立？"

### 16. One-Class SVM（单类 SVM — 异常检测）

**核心思想**：训练时只给模型看「正常样本」，让模型学习"正常数据长什么样"。测试时，如果来了一个不像正常样本的数据，模型就判定为异常。

> 去问 GPT："One-Class SVM 和普通的 SVM 有什么区别？nu 参数控制什么？"

### 17. KMeans（K 均值聚类）

**核心思想**：
1. 随机选 K 个中心点
2. 每个样本归属最近的中心点
3. 重新计算每个簇的中心
4. 重复 2-3 步直到稳定

看起来很简单，但在很多实际场景中效果出奇地好。

**如何选 K？**：
- **Elbow Method（肘部法则）**：画一条 K vs 误差的曲线，找拐点
- **Silhouette Score（轮廓系数）**：值越高说明簇内紧密、簇间分离

> 去问 GPT："KMeans 的 Elbow Method 为什么叫'肘部'？KMeans 有什么局限性？"

### 18. 层次聚类（Agglomerative Clustering）

**核心思想**：从每个样本自己是一类开始，逐步把最近的两类合并，直到只剩 K 类。结果可以用**树状图（dendrogram）**可视化，像一棵倒长的树。

**和 KMeans 的区别**：
- KMeans：速度快，但需要预先指定 K，假设簇是球形的
- 层次聚类：不需要预先指定 K（可以后期从树状图切），能发现任意形状的簇，但数据量大时会很慢

> 去问 GPT："层次聚类的 Ward 链接方法是什么意思？它和 Single Linkage 有什么区别？"

### 19. PCA（主成分分析）

**核心思想**：把高维数据压缩到低维，同时尽量保留原始数据的信息。比如 13 维 → 2 维后你还能在平面上画散点图，肉眼看到数据的分布。

> 去问 GPT："PCA 的数学直觉是什么？为什么它选择方差最大的方向？"

## 第四部分：评估指标 — 怎么判断模型好不好？

### 20. 准确率（Accuracy）

`正确预测的数量 / 总数量`

看起来直观，但在**类别不平衡**时是危险的指标。比如 1000 个人中只有 1 个患病，模型什么都不学直接预测"所有人健康"，准确率也是 99.9%——但这毫无用处。

> 问 GPT：为什么类别不平衡时 Accuracy 不可靠？

### 21. 混淆矩阵（Confusion Matrix）

一张 2×2 或 n×n 的表格，直观展示分类模型在每个类别上对了多少、错了多少：

|  | 预测：健康 | 预测：患病 |
|---|---|---|
| **真实：健康** | TN（真阴性）✅ | FP（假阳性）❌ |
| **真实：患病** | FN（假阴性）⚠️最危险 | TP（真阳性）✅ |

四个象限各有含义，其中 FN（有病但预测为没病）通常是医疗场景中最不能容忍的错误。

### 22. Precision（精确率）和 Recall（召回率）

- **Precision = TP / (TP + FP)**：模型说"患病"的人中，真正患病的比例。高了说明不会乱报警。
- **Recall = TP / (TP + FN)**：所有真正患病的人中，模型找出了多少。高了说明不会漏诊。

**Precision 和 Recall 往往互斥**——提升一个通常会牺牲另一个。实际项目中根据成本权衡：是宁可误报（抓错了人）还是宁可漏报（放过了坏人）？

> 问 GPT：Precision 和 Recall 为什么存在此消彼长的关系？

### 23. ROC 曲线 和 AUC

- **ROC 曲线**：横轴是假阳性率（误报），纵轴是真阳性率（召回），曲线越靠近左上角越好。
- **AUC**：ROC 曲线下的面积，0.5=随机猜，1.0=完美分类。AUC 不受类别不平衡影响，是比 Accuracy 更可靠的指标。

> 问 GPT：AUC=0.8 到底意味着什么？能翻译成人话吗？

### 24. R²、MSE、RMSE、MAE（回归指标的四个兄弟）

| 指标 | 一句话解释 | 越小越好/越大越好 |
|---|---|---|
| R² | 模型解释了多少比例的变异，1.0 是完美 | 越大越好 |
| MSE | 预测值和真实值差值的平方平均 | 越小越好 |
| RMSE | MSE 的平方根，量纲和原始数据一致，更直观 | 越小越好 |
| MAE | 差值的绝对值平均，对异常值不敏感 | 越小越好 |

> 问 GPT：R²=0.58 是什么水平？算好还是算差？

### 25. ARI 和 Silhouette Score（聚类评估）

- **ARI（调整兰德指数）**：看聚类结果和真实标签有多接近。1.0 是完美，0 是随机。
- **Silhouette Score（轮廓系数）**：不需要真实标签，看簇内是否紧密、簇间是否分离。

> 问 GPT：为什么评估聚类比评估分类困难得多？

## 第五部分：进阶工具与技术

### 26. GridSearchCV（网格搜索交叉验证）

**它解决的问题**：模型有一些手动设定的参数（如树的深度、学习率），到底设多大最好？

**工作原理**：你给一个参数范围（比如 max_depth 可能是 [3, 5, 7, 10]），它自动把所有组合都试一遍，用交叉验证评估每种组合，最后告诉你最佳参数。

> 问 GPT：GridSearchCV 中的 CV 是什么意思？交叉验证怎么做的？

### 27. SHAP（特征归因分析）

**它解决的问题**：一个模型预测"这个人会违约"，为什么？哪些特征对这个预测贡献最大？

SHAP 基于博弈论中的 Shapley 值，给每个特征一个"贡献分"——正分代表把预测推向正类，负分代表推向负类。比 RF 自带的 feature_importances_ 更精细、更统一。

> 问 GPT：SHAP 值和普通的 feature importance 有什么本质区别？SHAP 为什么被称为"模型解释的黄金标准"？

### 28. 过拟合（Overfitting）

**一句话**：模型在训练数据上考满分，一到考试就翻车。因为它是死记硬背而不是真正理解。

**常见信号**：训练准确率远高于测试准确率（如训练 99%，测试 85%）。

**解决方法**：减少模型复杂度、加正则化、增加训练数据、早停（early stopping）。

> 问 GPT：过拟合和欠拟合怎么判断？正则化为什么能防止过拟合？

### 29. 类别不平衡（Class Imbalance）

**一句话**：正样本和负样本数量差太多。比如 10,000 条数据中只有 200 条是违约的，模型会"偷懒"——直接预测所有人都不会违约，准确率也有 98%。

**解决方法**：SMOTE（生成少数类样本）、class_weight（惩罚误判少数类的错误更重）、降采样/过采样。

> 问 GPT：SMOTE 是怎么"生成"新样本的？这不就是在捏造数据吗？
